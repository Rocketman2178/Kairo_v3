## Security Advisories
- **6 SECURITY DEFINER views** (pre-existing): `v_test_conversations`, `organization_settings_view`, `full_session_details_view`, `available_sessions_view`, `session_recommendations_view`, `v_conversation_history`. These enforce view-creator permissions. Low risk for internal views but should be reviewed against Supabase recommendation at [docs](https://supabase.com/docs/guides/database/database-linter?lint=0010_security_definer_view).
- **Permissive INSERT policies** (`conversations`, `session_interest`, `tiger_tank_feedback`): Intentional — all three serve unauthenticated public flows (chat, notify-me, feedback).
- **`competitive_recommendations`, `regions`, `seasons`**: ALL policy with USING(true) for authenticated — overly broad, pre-existing.
- **Leaked password protection disabled**: Supabase HaveIBeenPwned check is not enabled. Low priority for demo but recommended before public launch.
- **26 functions with mutable search_path**: Pre-existing. Low risk in controlled environment; should add `SET search_path = public` to each for production hardening.

## Performance Advisories
- **auth_rls_initplan on `waitlist`, `sessions`, `locations`, `registrations`**: RLS policies call `auth.uid()` per-row instead of `(select auth.uid())`. Pre-existing pattern across the codebase. Should be fixed before high-volume use — replace `auth.<function>()` with `(select auth.<function>())` in RLS policies.
- **`waitlist.child_id_fkey` unindexed**: Add `CREATE INDEX idx_waitlist_child ON waitlist(child_id)` if waitlist grows.
- **`sessions.season_id_fkey` unindexed**: Add index on `sessions(season_id)`.
- **50 unused indexes** (INFO): Expected for dev database; indexes haven't been exercised yet. Monitor post-launch.
- **Multiple permissive policies** on `registrations`, `sessions`, `waitlist`, `locations`: Pre-existing, WARN level. Not a blocker but increases policy evaluation overhead.
