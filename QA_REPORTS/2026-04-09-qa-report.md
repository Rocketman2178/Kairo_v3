## Issues Found & Resolved

### BUG FIXED: Race condition in redeem-makeup-token enrolled_count increment

Severity: Medium
File: supabase/functions/redeem-makeup-token/index.ts

Root cause: The function called increment_session_enrolled_count RPC (does not exist in DB).
This triggered a catch block with a non-atomic fallback:
  enrolled_count: session.enrolled_count + 1
Since session was fetched before the registration insert, two concurrent requests could both
read the same enrolled_count, both pass the capacity check, and both write old_count+1
instead of old_count+2 — silently under-counting enrollment and allowing over-capacity booking.

The update_enrollment_count_on_registration trigger already handles this atomically via COUNT(*).

Fix: Removed entire step 6 increment block. Trigger handles it correctly.
Deployed: redeem-makeup-token v2 deployed to Supabase.
