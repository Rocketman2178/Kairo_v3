## Issues Found & Resolved

### BUG FIXED: `calculate_registration_discount` — Broken Column References

**Severity:** High — function throws error on every call, breaking discount calculation for all registrations.

**Root cause:** Function body referenced two columns that don't exist on the `organizations` table:
- `returning_family_discount_percent` → actual column: `returning_discount_percent`
- `early_bird_days_before_start` → actual column: `early_bird_days`

**Fix:** Applied migration `fix_calculate_registration_discount_column_refs` — replaced wrong column names with correct ones. Also added `SET search_path = public` to harden against mutable search_path advisory.

**Verification:** Function now executes correctly — returns `{discount_percent: 5, discount_reason: "Early bird discount", discount_amount_cents: 1040}` for a sample confirmed registration.

### BUG FIXED: `get_available_transfer_sessions` — Non-existent Column Reference

**Severity:** High — function would fail immediately on any call (new feature, not yet in use).

**Root cause:** Function referenced `s.organization_id` on the sessions table, which doesn't have that column. Sessions link to organization via `programs.organization_id`.

**Fix:** Updated migration file (`20260408000001_add_class_transfers.sql`) before applying:
- Line 87-92: Added `JOIN public.programs p ON p.id = s.program_id` and changed `s.organization_id` → `p.organization_id`
- Line 116: Changed `WHERE s.organization_id = v_org_id` → `WHERE p.organization_id = v_org_id`

**Verification:** `get_available_transfer_sessions()` returns correct results for a confirmed registration.
